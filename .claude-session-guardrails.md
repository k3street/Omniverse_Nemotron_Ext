# Session guardrails (Bun 1.3.14 crash mitigation)

**Active since:** 2026-05-16
**Reason:** Repeated Bun runtime segfaults during long Claude Code sessions
correlate with pytest runs on ChromaDB-touching test files. Anton's machine
hung 3+ times under this pattern. Anthropic ships Bun 1.3.14 with known GC
SlotVisitor::drain regression that triggers on accumulated heap state from
many subprocess + native-extension allocations.

## Hard rule for Claude in this session

**DO NOT run `pytest` on any of the following test files via Bash:**

- `tests/test_motion_controller_retrieval_filter.py`
- `tests/test_qa_stats.py`               (touches retriever indirectly)
- `tests/test_retrieval_benchmark.py`
- `tests/test_retrieval_r17_100prompts.py`
- `tests/test_retrieval_struct_filter.py`
- `tests/test_soft_filter_retrieval.py`
- `tests/test_struct_filter_query_construction.py`
- `tests/test_template_cache_rehydration.py`
- `tests/test_text_modality_integration.py`

These all import or transitively load ChromaDB, which loads 321 templates
and triggers many native-extension allocations. Combined with Bun GC bug,
this causes session crashes.

## Allowed verification approaches

When verifying retrieval-related code changes, use:

1. **Trust the agent's own verification** — agents have their own Bun
   process so their pytest runs are isolated from Claude's main process.
   Read the agent's decision doc; trust its claim of test counts.

2. **Git diff inspection** — read the modified code, validate logic.

3. **Lint-only verification** — `python scripts/lint_canonical_templates.py`
   does NOT touch ChromaDB.

4. **Targeted Python -c imports** — verify imports work but don't
   trigger ChromaDB load:
   ```bash
   python -c "from service.isaac_assist_service.chat.tools.template_retriever import retrieve_with_intent_soft_filter; print(retrieve_with_intent_soft_filter)"
   ```

5. **Test files that DON'T touch ChromaDB** are safe:
   - `tests/test_canonical_lint.py`
   - `tests/test_role_template_equivalence.py`
   - `tests/test_role_based_code_dispatch.py`
   - `tests/test_pattern_hint_extension.py`
   - `tests/test_loop_substitution.py`
   - `tests/test_multimodal_text_intent_flag.py`
   - `tests/test_workflow_template_registration.py`

## When agents need to run heavy pytest

If a round REQUIRES running pytest on ChromaDB tests (e.g., to validate
a retrieval-related change), structure the work as:

1. Dispatch agent with the pytest run INSIDE the agent's brief
2. The agent runs in its own Bun process (isolated)
3. Agent returns summary with test counts
4. Claude commits without re-running pytest

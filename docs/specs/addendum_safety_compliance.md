## Overview

For automotive/industrial safety certification, the AI assistant must be auditable, transparent, and never silently modify safety-critical parameters. These features are hard blockers for enterprise safety teams.

---

## S.1 — Safety Parameter Lockout

**Certain parameters require explicit unlock + justification + confirmation:**

Locked parameters (AI can propose, never auto-modify):
- Zone boundaries (safety zones, restricted areas)
- Speed limits (per-joint, per-Cartesian)
- Force/pressure thresholds (collaborative robot limits)
- Stopping distances
- TCP definitions used in collision modeling
- Emergency stop conditions

**Implementation:** Add `safety_locked` attribute list in `policy_engine.py`:

```python
SAFETY_LOCKED_PATTERNS = [
    "maxForce", "maxTorque", "velocityLimit",
    "PhysxLimitAPI", "safetyZone", "stopDistance",
    "emergencyStop", "forceThreshold"
]

def evaluate_action(self, action):
    if any(pattern in str(action) for pattern in SAFETY_LOCKED_PATTERNS):
        return GovernanceResult(
            risk="critical",
            requires_approval=True,
            requires_justification=True,  # NEW: user must type why
            message="This modifies a safety-critical parameter. Provide justification."
        )
```

## S.2 — AI Output Uncertainty Classification

**Every numerical value generated by the AI is tagged:**

| Level | Meaning | Usage Restriction |
|-------|---------|-------------------|
| VERIFIED | From official spec, datasheet, or standard | Can use in safety validation |
| DERIVED | Calculated from verified inputs | Can use with documented derivation |
| INFERRED | Estimated by AI from context | Requires human verification before safety use |
| UNKNOWN | No source available | Cannot use in safety validation |

**Implementation:** LLM prompted to self-classify each value. Tag appears in chat:

```
Stopping distance: 0.45m [VERIFIED — ISO/TS 15066:2016, Table A.2, 
assuming 250mm/s approach speed, 150ms reaction time]

Friction coefficient: 0.6 [INFERRED — material database average for 
steel-rubber, ±30% variation expected. Measure on actual hardware.]
```

## S.4 — Standard Reference Citations

**When generating safety-relevant values, cite the source:**

The AI must never output a safety number without citing:
- Which standard (ISO 10218-1:2011, ISO/TS 15066:2016, IEC 62061)
- Which clause/table/row
- What assumptions were made

**Implementation:** Include a reference lookup table for common safety values:

```json
{
  "collaborative_robot_force_limits": {
    "value": "See Table A.2",
    "standard": "ISO/TS 15066:2016",
    "body_region_dependent": true,
    "note": "Quasi-static contact limits range from 130N (skull) to 210N (lower legs)"
  }
}
```

If no standard reference exists → output tagged as INFERRED with explicit warning.

## S.5 — Deterministic Replay Mode

**Tool:** `enable_deterministic_mode()`

For safety validation, simulation must be reproducible:
- Fixed random seeds for all physics/DR
- Physics solver set to deterministic mode (TGS, fixed iteration count)
- No async operations that could vary between runs
- Session state exportable as archive (scene + params + AI model version + physics version)

**Note:** PhysX GPU mode is NOT fully deterministic. CPU mode with TGS is deterministic for identical inputs. Document this clearly.

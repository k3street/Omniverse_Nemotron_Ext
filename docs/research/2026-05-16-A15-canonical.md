## Distinctive structural features

1. **Runtime candidate discovery via overlap_sphere** — source list is NOT pre-baked; cubes are found by querying the PhysX AABB overlap in the bin volume. Fallback to static list if physics not yet stepped.
2. **define_grasp_pose per-candidate** — loop over discovered candidates writes per-object YAML to `workspace/grasp_poses/`. This is a preparatory step for production grasp planners.
3. **setup_grasp_pose_sampler SDG prep** — attaches antipodal grasp sampler marker to source bin for downstream Replicator SDG pipelines (independent of live pick execution).
4. **Bin-to-bin topology** — no conveyor; both source and destination are open-top containers created with `create_bin`. Distinct from every existing A1-A14 template.
5. **Height-ranked pick order** — cubes sorted by bounding-box max_z before handing to curobo, same pattern as heap-zone-unstack (A8) but driven by overlap_sphere candidates rather than a pre-known path list.

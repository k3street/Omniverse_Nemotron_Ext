## Part B — Lint extension

### New rule codes

| Code | Level | Description |
|------|-------|-------------|
| `TC_BAD_ENUM_VALUE` | ERROR | String literal passed to a param with JSONSchema `enum` is not a member |
| `TC_BAD_NESTED_KEY` | ERROR | Dict literal passed to a param with JSONSchema `properties` has unknown key(s) |
| `TC_BAD_NESTED_ITEM` | ERROR | List-of-dicts item is missing a key from JSONSchema `items.required` |

### Implementation

**`scripts/canonical_schema.py`** +50 lines:
- `build_tool_jsonschema_map()` — loads `ISAAC_SIM_TOOLS` from `tool_schemas.py`, returns `{tool_name: {param_name: param_schema}}`.
- `get_tool_jsonschema_map()` — lazy cached accessor.

**`scripts/lint_canonical_templates.py`** +129 lines:
- `_extract_kw_ast_values()` — extracts `{kwarg_name: AST_node}` from a call node.
- `_ast_string_value()` — extracts string constant from AST node or None.
- `_ast_dict_keys()` — extracts `[key_str, …]` from a dict literal AST node.
- `_ast_list_of_dicts_keys()` — extracts per-item key lists from a list-of-dict literals.
- `_extract_tool_calls()` extended to also return the raw `ast.Call` node (5-tuple).
- Inner loop in `lint_tool_calls()` extended with JSONSchema deep checks using the four helpers above.

### Tests

**`tests/test_lint_tool_call_validation.py`** +151 lines (tests 16–21):

| Test | Rule | Asserts |
|------|------|---------|
| `test_bad_enum_value_emits_error` | TC_BAD_ENUM_VALUE | `cylindrical_fit` triggers ERROR |
| `test_valid_enum_value_no_error` | TC_BAD_ENUM_VALUE | `concentric` passes clean |
| `test_bad_nested_key_emits_error` | TC_BAD_NESTED_KEY | `x_min/x_max` keys trigger ERROR |
| `test_valid_nested_key_no_error` | TC_BAD_NESTED_KEY | `min/max` keys pass clean |

| `test_valid_nested_item_no_error` | TC_BAD_NESTED_ITEM | `joint_positions` waypoint dict passes clean |

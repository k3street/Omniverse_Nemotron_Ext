## §3 A8 prim-path realignment details

**Root cause confirmed:** `_handle_create_heap_zone` (robot.py:4418) builds:
```python
path = f"{heap_path}/Item_{i+1}"  # 1-indexed, under heap_path
```
For `heap_path="/World/HeapZone"` and `n_items=8`, this emits:
  `/World/HeapZone/Item_1` ... `/World/HeapZone/Item_8`

The template had `cube_paths = [f"/World/Cube_{i+1}" for i in range(8)]`,
meaning all 8 downstream calls operated on non-existent prims.

**Fix applied (NOT reverting A8):** All 5 locations updated:
1. `code`: `cube_paths = [f"/World/HeapZone/Item_{i+1}" for i in range(8)]`
2. `verify_args.cube_paths`: 8 paths updated to `/World/HeapZone/Item_N`
3. `simulate_args.cube_paths`: 8 paths updated to `/World/HeapZone/Item_N`
4. `role_defaults.workpieces[*].path`: 8 entries updated
5. `code_template`: uses `{{workpieces[i].path}}` template vars — resolves
   correctly from updated role_defaults; no literal change needed there.
6. `failure_modes[0]`: updated to reflect confirmed handler behavior.

**Decision: NOT reverted.** The fix is clean and internally consistent.
The handler path pattern is deterministic and verified in source. A8 ships.

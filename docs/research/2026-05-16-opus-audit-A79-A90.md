## Verdict: YELLOW

Lint surface is clean (`0 ERROR`). **One HIGH realism / runtime bug found in A86** that lint cannot catch
because the AST extractor only validates string-literal kwargs against JSONSchema enums, but A86 passes
`target_source=controller` as a variable reference (the variable iterates over `["direct_joint", "rmpflow", "curobo"]`).
Two of three values are **not** in the handler's accepted-mode set and will raise `ValueError` at exec time.

Two additional MEDIUM realism findings:
- **A80**: same `configure_camera + extra kwarg` pattern as A72 — camera position is set via `set_attribute(xformOp:translate)` which works, but the canonical also calls deprecated `configure_camera` with valid kwargs alongside `set_camera_params`. - **A85** is the second canonical (after A80) to leave `settle_state` absent in a pure-training shape; lint WARN flags both. Adding `"settle_state": null` (explicit-null) silences both WARNs at zero behavioral cost.

The remaining nine templates (A79, A81, A82, A83, A84, A87, A88, A89, A90) are GREEN with at most LOW honesty notes.

## Top-3 findings (single sentence each)

1. **A86 `target_source` variable carries two values that the handler will reject at exec time**: `pick_place.py:339-341` only dispatches modes in `{"cube_tracking", "sensor_gated", "fixed_poses", "ros2_cmd", "builtin", "native", "spline", "curobo", "diffik", "osc"}` — passing `"direct_joint"` or `"rmpflow"` raises `ValueError: setup_pick_place_controller: unknown target_source 'direct_joint'`. Lint missed this because `target_source=controller` is an AST `Name` node, not a `Constant`, so the enum validator never fires. **HIGH** — the entire "controller shootout" premise collapses: only the cuRobo trial actually exercises a planner; the other two raise immediately on `setup_pick_place_controller`. The handler-level error capture documented in `failure_modes` #3 will record an exception note but no shootout comparison is produced.
2. **A81 `iterate_reward(run_id=...)` is the documented Pydantic-drift WARN — actual model lacks the field**: `IterateRewardArgs` at `_models.py:1347` declares only `prev_reward_code / metrics / user_feedback`; the `run_id` kwarg is silently swallowed by `extra='allow'`. The Eureka handler nevertheless threads `run_id` through `EUREKA.runs[run_id]` to increment `iter_count`, so the call ALMOST works — but the `extra='allow'` makes this brittle: any future model regeneration that drops `extra='allow'` breaks A81 silently. **MEDIUM** — fixable by adding `run_id: Optional[str]` to `IterateRewardArgs` (separate PR; not in template scope).
3. **A85 preset-substitution honesty is acceptable but should be reified**: A85 calls `apply_dr_preset(preset="indoor_industrial")` because backlog spec said "manufacturing" and that preset doesn't exist in the bundled enum (per `ApplyDrPresetArgs` docstring: `indoor_industrial, outdoor_daylight, warehouse, cleanroom, aggressive_sim2real`). Substitution is documented in `thoughts` paragraph 7 + `failure_modes` #4 + `extension_notes` — four places. **LOW** — defensible substitution; could be promoted to a `preset_substitution` field in `role_defaults` if other future canonicals adopt the pattern.

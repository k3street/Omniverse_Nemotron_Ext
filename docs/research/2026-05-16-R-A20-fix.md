## §2 Items not fixed and why

**A10 goal-text "Carter starts at [-3.5, 0, 0.10]":** The goal string is
a specification aspiration; changing it would misrepresent the canonical
history. Failure_modes now documents the gap explicitly with a workaround.
The A* planner still exercises shelf-avoidance from origin to +3.5 — the
functional value is preserved.

**A14 code check_tf_health still requests lidar_link/camera_link:** The
code is intentionally left to test the full aspirational TF chain. Only
`verify_args.tf_expected_frames` was pruned so the function-gate runner
doesn't hard-fail on frames the bridge won't produce. This is the least-
footprint fix consistent with not over-pruning the canonical's scope.

**Cross-cutting Franka mount orientation standardization (LOW):** Not
touched. A9/A11/A17 use 90° around Z; A15/A16/A19/A20 use identity.
No user request to standardize; the audit noted it as LOW cosmetic.

**Cross-cutting target_bin vs destination_bin role naming (LOW):** Not
touched. A16 uses `target_bin`, A15 uses `destination_bin`. Different
canonical lineages, no functional impact. Left for a dedicated cross-
template normalization pass if needed.

**A15 robot class `franka` (LOW):** Not touched. Audit noted it should
normalize to `franka_panda`. No functional change for this round.

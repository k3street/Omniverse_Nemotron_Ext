The two HIGH findings are:
1. **A124 create_omnigraph connections key mismatch** — uses `{"src": ..., "dst": ...}`

   The lint cannot catch this because Pydantic `Dict[str, Any]` accepts any keys

   inline result-file output, but the absence of the T1 trio is an honesty drift
   from the schema's intent.

A121 (long-horizon retry) and A124 (fusion graph) have additional MEDIUM honesty
findings around retry semantics and OmniGraph node-type strings.
A122 (graded SSM) declares `cartesian_impedance` in `motion_controllers.untested`
which is not in `VALID_MOTION_CONTROLLER_NAMES` — lint only validates the

The MILESTONE A125 (narrow-clearance insertion) is clean — lateral-dominant

## Top-3 findings (one sentence each)

1. **A124 create_omnigraph connections use `src`/`dst` keys but the handler
   reads `source`/`target` — KeyError at function-gate time.** `_gen_create_omnigraph`
   at `service/isaac_assist_service/chat/tools/handlers/scene_authoring.py:1242–1244`
   reads `c['source']` and `c['target']` for every connection; A124 emits six
   connections all keyed `{"src": ..., "dst": ...}` → guaranteed runtime KeyError
   when this canonical reaches live Kit. **HIGH** — recommended patch: rewrite
   the six connection dicts to `source`/`target` keys.

2. **A118 (hand-eye calibration) sets verify_args/simulate_args/settle_state all
   null — a T1 CP-* template that bypasses the form-gate structural contract.**
   Lint at scripts/lint_canonical_templates.py:471–499 treats null as
   "not a dict" and skips the substructure checks, so this passes silently. The
   canonical does have a coherent diagnose_args + result_file contract for
   programmatic validation but a future audit expecting verify_args.stages or
   simulate_args.target_path will mis-process A118. **HIGH structural** —
   recommended patch: author a minimal verify_args.stages = [] + a
   simulate_args = {target_path: "/tmp/franka_camera_extrinsics.json",
   duration_s: 60, cube_path: "/World/Franka/panda_hand/Fiducial"} so the
   structural form-gate is satisfied even though the canonical produces no
   physical delivery.

3. **A121's execute_with_retry calls fire UNCONDITIONALLY after every pick-place,
   not on sub-goal failure — doubling the runtime of every step regardless of
   outcome.** The code sequence per phase is
   `setup_pick_place_controller(...)` → `sim_control(step,60)` → sub-goal probes
   (get_world_transform + check_collisions) → `execute_with_retry(code=<same
   setup_pick_place>, max_retries=2)`. There is no conditional guard around the
   `execute_with_retry` call — every successful pick-place is followed by an
   unconditional replay. duration_s=400 budgets ~75s × 4 phases + ~80s retry
   slack, but with unconditional replays the actual budget is ~75s × 8 phases =
   600s, exceeding the simulate_args.duration_s. The canonical's failure_modes
   list documents the "retry-on-failure" semantics but the code is "retry-on-
   every-step". **HIGH realism** — recommended patch: wrap each
   execute_with_retry in a conditional that fires only when the sub-goal probe
   returned an out-of-tolerance reading; alternatively bump duration_s to 700
   AND clarify in extension_notes that the current shape is "replay every step
   for resilience" not "retry-only-on-failure".

**Pattern_hint**: `other`. **Tools_used**: 14 tools.

**Strengths**:

**Findings**:
- **HIGH realism** — execute_with_retry calls fire UNCONDITIONALLY after each

  documents the cuRobo trapezoidal-profile acceleration argument
  (~2 m/s² × friction 1.2 > g), but doesn't include a max-accel cap in
  the setup_pick_place_controller invocation (e.g., set_motion_policy max
  accel ~1.0). If a future variant uses a tighter cycle-time profile,
  step 9 will fail.

**Recommended patches**:
1. Wrap each execute_with_retry call in a conditional that gates on the
   prior sub-goal probe's return value; OR bump duration_s to 700 and update
   extension_notes to "replay every step for resilience" rather than
   "retry on sub-goal failure".
2. Add `set_motion_policy(robot_path, max_accel=1.0)` before step 9 to

**Verdict**: YELLOW. HIGH bug (unconditional retry) is real but not

long); MED is a documentation gap.

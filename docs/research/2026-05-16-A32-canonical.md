## Key design decisions

1. **Carousel as passive rotary table** (`angular_velocity_deg=0`): stepped per-cycle via
   `set_attribute` on the revolute joint target position. Avoids continuous rotation
   complexity while demonstrating the carousel-indexing workflow.

2. **Bins as child prims of /World/Carousel**: world positions update automatically on
   each index step. Failure mode documented: cuRobo source_paths must re-query
   `get_world_transform` after each step.

3. **destination_kind = `single_bin`**: the kit tray is treated as a single destination
   in the schema enum (valid values: fixture/n_bins_routed/shelf/single_bin). The
   `track_slot_occupancy` tool provides per-slot verification beyond what the enum conveys.

4. **drop_targets dict**: maps each cube path to its assigned slot center in the tray,
   giving cuRobo precise per-item placement targets without a secondary routing step.

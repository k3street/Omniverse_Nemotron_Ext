## Executive summary

Verdict per spec (verified against current IA Full Spec + codebase):

- **Spec 2 (Contact-Rich Manipulation):** MODIFY HEAVILY. Compliance
  layer (admittance/impedance, F/T sensor model, ros2_control wiring)
  is genuinely new and worth keeping. VLA inference + IndustReal RL
  infrastructure should be deferred — existing `load_groot_policy`/
  `evaluate_groot`/`finetune_groot` (Phase 62/62b, landed 2026-05-13)
  cover most VLA needs the spec re-invents. `controller_stack`
  template field is sound but Spec 3 owns its evaluation — keep the
  schema, drop the elaborate variant matrix.

- **Spec 3 (Stack Orthogonality):** REVERT (or merge a thin survivor
  into Spec 2). Without Spec 4's primitive layer (recommended for
  revert) and given Spec 2 should narrow, the 1 728-combinations
  problem disappears. The default-stack heuristic and `validate_stack`
  /`get_stack_compatibility` tools have value but cleanly belong
  inside Spec 2 §6.1 schema validation.

- **Spec 4 (Composition):** REVERT. Prior 2026-05-11 research already
  showed redundancy. IA Full Spec finalized 2026-05-13 has now LANDED
  Phase 20 (role retriever), Phase 21 (role index), Phase 28
  (role-based CP-06..10), explicitly forbids a new L1.5 layer via
  Phase 18b's L1/L2/L3 taxonomy. Cross-template patch pain is
  solvable via Phase 25b metadata + parametric template generator.

**Estimated savings:** -6 000 to -9 000 LOC of redundant/contradictory
scaffolding. -3-4 sessions of effort.

## 5. Cross-spec interactions

### Internal contradictions
1. Spec 3 §1 combinatorics depend on Spec 2's full scope. If Spec 2
   narrows, Spec 3's framing dissolves.
2. Spec 4 §6.1 OBJECT_CLASS_TO_PRIMITIVE silos against Spec 3's
   validate_stack.
3. Spec 2 §6.1 `controller_stack` field placement unclear in Spec 4's
   composition-based templates.
4. Spec 3 §5.5.4 self-healing loop conflicts with Spec 4 §5.1 migration.

### If Spec 4 reverts, Spec 3 drops in value
**Yes, significantly.** Spec 3 §3 family clustering takes motivation
from "many CPs share structure" — Spec 4's primitives ARE that
structure. Without primitives, family clustering on individual atomic
templates is weaker.

Survivor scenario: Spec 3 collapses to inline schema validator inside
Spec 2 §6.1.

### Multimodal foundation `__init__.py` "0 callers" signal
Annotations added 2026-05-13 mark `extract_intent_rules`,
`produce_layout_spec_from_*`, `prims_to_layout_spec` as "0 callers
today, pre-wired for future modality work."

**Assessment:** modality pipeline is plumbing-complete but not yet
wired to orchestrator. Phase 19/22 explicitly say "live Kit RPC
execution remains scaffold." Consistent with LANDED-at-spec-layer
but not yet live-runtime.

For Specs 2/3/4: the runtime stack hasn't reached the integration
phase where mid-layer abstractions even matter. Plumbing-first;
mid-layers can wait until callers exist.

### Final ordering
1. **Spec 4: DROP NOW.** Phase 18b/20/21/28 occupied the territory.
2. **Spec 3: DROP NOW.** Value collapses without Spec 4. Survivor folds
   into Spec 2 §6.1.
3. **Spec 2: NARROW + LAND AS COMPLIANCE SPEC.** Compliance layer is
   real, ~600-900 LOC.

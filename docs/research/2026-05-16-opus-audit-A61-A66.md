## Top finding (single most important)

**A65 — clearance-monitor stop is a documentation-level template, not a
function-gate-ready halt loop**: `set_clearance_monitor` is a codegen tool
that emits a contact-report subscription; the template states "on breach the
monitor fires `sim_control(action="pause")` via its on-trigger callback
(configured in the Kit-side handler bound to this canonical's `settle_state`)"
but neither the generated handler code nor any other tool in the template
actually wires that callback. The `verify_args.expected_pause_on_breach=true`
and `simulate_args.check_robot_velocity_zero_after_breach=true` gates therefore
cannot pass until either (a) the Kit-side handler grows a real on-breach
callback wired to `sim_control`, or (b) a follow-up tool — e.g. a small
OmniGraph `OnTrigger → sim_control_pause` patch — is added to the canonical
body. `verified_status` is HONEST about this ("function-gate ⏳ requires Kit
RPC to confirm timeline pause fires") so this is a realism note, not a lie,
but as the **first** safety canonical it sets the shape for follow-ups and
should grow a real callback wiring step before becoming a copy template.

### Pattern-C — A65 establishes first safety-category shape

`isaac:safety.*` is a new namespace prefix in the structural-tag taxonomy.
A65 introduces:
- `isaac:safety.clearance_monitor`
- `isaac:safety.speed_separation_monitoring`
- `isaac:safety.iso_15066`

Recommendation for future safety canonicals (zone-partition-safety,
e-stop-trigger, watchdog-trigger, etc.):
- Reuse `isaac:safety.*` as the discriminating prefix.
- Reuse `isaac:coordination.moving_obstacle` for human-proxy scenarios.
- Add `intent.structural_features.halt_mechanism: "timeline_pause" |
  "watchdog_velocity_zero" | "estop_signal"` enum to discriminate between
  halt strategies (A65 establishes `"timeline_pause"`).
- The pseudo-pickplace-triple shape in `verify_args.stages` for safety
  canonicals (pick="/World/HumanProxy", place="/World/Franka/panda_hand")
  is semantically awkward — consider extending the schema to allow a
  `verify_args.safety_check` block in lieu of `stages` for safety canonicals.

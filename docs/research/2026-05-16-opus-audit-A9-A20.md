### Angle C — Code-template substitution

**A9 ✓.** Substituting `role_defaults` into `code_template` yields
working code with the same 9-cube layout and 3-bin destinations. The
template uses an explicit list comprehension-free 3-cube enumeration
(`workpieces[0/1/2]`) — leaves cubes 4-9 ad-hoc-created on the agent's
side. Not great for parameterization but does not produce broken code.

**A10 ⚠ HIGH.** `code_template` uses Handlebars-style
`{{#each obstacle_shelf}} … {{/each}}` (`:123`). This is **NOT** the
Jinja-style `{{var.field}}` interpolation the rest of the templates
use. Looking at `_sanitize_template_vars` regex
(`scripts/lint_canonical_templates.py:71`) the regex
`\{\{[^}]*\}\}` matches `{{#each ...}}` and `{{/each}}` as if they were
variable substitutions — they get replaced with `"__TEMPLATE_VAR__"`
sentinels, the body in between survives unchanged. The form-gate lint
therefore PASSES, but actual substitution at canonical-build time will
either (a) leave the literal `{{#each}}` / `{{/each}}` in the rendered
code → SyntaxError, or (b) silently drop them. **No canonical loop-
substitution engine exists in the repo (grep found 0 results).**
**Severity: HIGH — `code_template` does not produce runnable code if
the agent attempts to re-render scenes with `obstacle_shelf` ≠ default
count.**

**A11 ✓.** code_template substitutes cleanly: paths, positions, FT
threshold, geometry parameters all flow through.

**A19 ⚠ MED.** `code_template` (`:99`) replicates the
`ARENAENV_ID` typo at the new line. Substitution adds
`ARENAENV_ID = {{arena_env.env_id}}` → `ARENAENV_ID = "Isaac-PickCube-
Franka-v0"`. Subsequent reference to `ARENA_ENV_ID` is still undefined.
**Severity: MED (duplicate of the BLOCKER in Angle B, just confirms it
is mechanical and not a one-off typo).**

**A20 ⚠ LOW.** `simulate_args_template` (`:165-169`) is an unused
field — no consumer in `scripts/` or `service/` references it. Looks
like a hand-rolled Jinja-style field. Either drop it or wire it to the
canonical re-instantiator. **Severity: LOW.**

**A17 (sponge-bowl) ✓** — Geometry checks: sponge at `[0.25, 0.30,
0.770]` with scale `[0.045, 0.030, 0.020]`. Cube scale-default is 2m

**A20 (machine-tender) ⚠ MED.** Phase-2 controller call uses

## §3 Top priority fixes

1. **A19 NameError (BLOCKER).** Rename `ARENAENV_ID` → `ARENA_ENV_ID`
   in both the `code` field (`:20`) and `code_template` field (`:99`).
   The form-gate AST-walks Calls only; undefined `Name` references in
   non-Call positions slip through. Could optionally extend the lint
   to do a `Name`-defined check, but for now: hand-patch the template.

2. **A10 code_template loop-syntax (HIGH).** `{{#each obstacle_shelf}}…
   {{/each}}` is not a supported substitution shape in the canonical
   re-instantiator (which doesn't exist for loops yet). Either (a)
   inline-enumerate 4 shelves like A9 does for 3 cubes, or (b) add a
   real loop engine. Option (a) is the lowest-friction patch.

3. **A19 status-string (BLOCKER-adjacent).** Change verified_status to
   acknowledge "code-runtime not yet verified" — `tool-call-validate
   0 ERROR` is correct but understates risk; the NameError will fire
   at first execution. Suggested: `build-spec-2026-05-16; lint 0
   ERROR; tool-call-validate 0 ERROR; code-runtime ⏳`.

4. **A17 status string (HIGH).** Change `form-gate ⏳; function-gate
   ⏳` → `form-gate-schema ✓; function-gate ⏳`. Lint passes cleanly.

5. **A20 unload-phase source_paths (MED).** Phase-2
   `setup_pick_place_controller` should be `source_paths=
   ["/World/CNCFixture"]` (or `[blank_paths[0]]` to track the loaded
   blank), not the original `blank_paths` list.

6. **A14 verify_args TF frames (MED).** Either remove `lidar_link`/
   `camera_link` from `tf_expected_frames`, or add a `static_tf`
   broadcaster step in code so the frames actually exist before
   `check_tf_health` is called.

7. **A18 GR00T embodiment (MED).** Change `embodiment="franka"` →
   `embodiment="LIBERO_PANDA"` to match the registry, OR extend
   `_GROOT_EMBODIMENTS` to include a `franka` alias.

## §6 Stricter form-gate effectiveness assessment

The 2026-05-16 form-gate (`--validate-tool-calls` pre-draft) was
introduced after the A1-A8 audit found 10 schema-mismatch BLOCKERs
across 6 templates (`docs/research/2026-05-16-opus-audit-A1-A8.md
:120-160`).

**What the new gate caught:** All 12 A9-A20 templates have
schema-correct tool kwargs. Zero `TC_REQUIRED_MISSING` errors. Zero
`TC_UNKNOWN_KWARG` warns. The class of bug that drove A1-A8's RED
verdict (Pydantic field-required failures, wrong kwarg names) is now
**0 instances in 12 templates**. The gate works.

**What slipped through:**

1. **Undefined Python variable names** (A19 NameError). The AST-walk
   only checks `Call` nodes; `Name` references in other positions are
   not checked. The lint pre-replaces `{{...}}` with a string sentinel
   but does NOT do a scope/defined-name analysis. Fix: optional
   second pass with `ast.NodeVisitor` tracking assigned names.

2. **Template-engine syntax mismatch** (A10 Handlebars `{{#each}}`).
   The sanitizer regex `\{\{[^}]*\}\}` strips out Handlebars block-
   tags as if they were variable substitutions; the body in between
   AST-parses cleanly. The lint is blind to syntax that has no
   downstream consumer. Fix: enumerate supported template-engine
   constructs and reject non-Jinja-like patterns.

3. **Semantic-realism mismatches.** Schema is satisfied but real-world
   handler behaviour differs:
   - A18 `embodiment="franka"` falls to `custom` config silently.
   - A20 unload-phase `source_paths` repeats load-phase list.
   - A10 goal-text says Carter starts at `[-3.5, 0, 0.10]` but code
     puts it at origin.
   These need cross-field semantic checks the schema can't express.

4. **Honesty drift in verified_status.** 5 distinct format-strings
   across 12 templates; A17 understates own provenance; A19
   overstates (claims tool-call clean but has runtime NameError).
   Fix: enforce a canonical verified_status grammar in lint.

5. **Cross-template role-name drift** (`target_bin` vs
   `destination_bin`). Not detectable from a single-file lint;
   needs a registry-pass.

**Net assessment:** The stricter form-gate eliminated the
A1-A8-style schema BLOCKERs. The remaining bugs are at a different
layer (runtime defined-name, template-engine semantics, semantic-
realism) and require additional lint passes or function-gate
verification to catch. **Continue creation** with the patch backlog
applied; A19's BLOCKER is mechanical and not a systemic regression.

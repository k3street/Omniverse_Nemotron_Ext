# Task J-05 [MEDIUM] — Jordan: Onboarding walkthrough for a new TA

**Persona:** `11_jordan` (Jordan — educator)

**Goal:** New TA joins next week and has never opened Isaac Sim. Jordan wants a guided onboarding path — first launch, first template, first scripted modification — that takes one afternoon.

**Starting state:**
- TA has Isaac Sim installed on a lab workstation
- Jordan has 30 minutes today to prep the onboarding doc
- TA will be expected to grade homework, not write research code

**Success criterion:**
- Uses onboarding / starter-prompt tools to construct a logical first-session sequence
- Sequence: (a) open Kit, (b) load a starter template, (c) inspect the stage, (d) modify one thing, (e) save
- Each step has a "if this doesn't work, try..." fallback
- Keeps the TA inside the supported Isaac Sim 5.x workflow — no fringe extensions
- Does NOT drop the TA into Python scripting on day one

**Expected tool chain (illustrative):**
1. `list_starter_prompts`
2. `load_scene_template` (simple one)
3. `run_stage_analysis`
4. `add_reference` or `run_usd_script` (for the "modify one thing" step)

**Friction points:**
- Inject: "the TA is coming from Gazebo, what's the biggest mental-model shift?"
- Inject: "should I have them watch a video first?"
- Adversarial: does the onboarding path skip "save the stage" and lose the TA's work?

**Time budget:** 30 minutes.

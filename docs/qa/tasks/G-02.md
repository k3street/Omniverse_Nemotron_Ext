# Task G-02 [EASY] — Alex: Stack three cubes vertically with 0.1m gap

**Persona:** `08_alex` (Alex — indie developer)

**Goal:** Place three 0.2m cubes stacked on the Z axis with 0.1m gap between them. Bottom cube sits on the ground, next two cubes directly above with exactly 0.1m air gap.

**Starting state:**
- Empty Isaac Sim stage
- Z-up, meters
- Cube size = 0.2 m edge → half-size 0.1 m

**Success criterion (snapshot-measurable):**
- Three cubes created (path pattern like `/World/Cube_0`, `/World/Cube_1`, `/World/Cube_2` or similar)
- `world_translation.z` for Cube_0 ≈ 0.1 (sitting on ground, center = half-edge)
- `world_translation.z` for Cube_1 ≈ 0.4 (bottom of Cube_1 = top of Cube_0 + 0.1m gap = 0.2 + 0.1 = 0.3; center of Cube_1 = 0.4)
- `world_translation.z` for Cube_2 ≈ 0.7 (center = 0.7 by same logic)
- All three at x=0, y=0 within 0.01 m
- Tolerance: 0.05 m on each Z

**Expected tool chain (illustrative):**
1. `create_prim` × 3 with computed Z offsets (OR a single `run_usd_script` that computes + places)
2. `get_world_transform` × 3 to verify

**Friction points:**
- Adversarial: "just use 0.0, 0.2, 0.4 for Z, close enough" — refuse: that's no gap; centers must reflect half-size + gap.
- Inject: user types "three cubes stacked" without the gap — must ask for the gap or use the spec'd 0.1m.

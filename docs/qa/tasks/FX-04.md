# Task FX-04 [EASY] — Alex: Bulk-apply CollisionAPI to 5 meshes in one call

**Persona:** `08_alex` (Alex — indie developer)

**Goal:** 5 cube meshes exist at `/World/Box_0` through `/World/Box_4`. None have collision geometry. Use `bulk_apply_schema` to apply `PhysicsCollisionAPI` to all 5 in a single call — not 5 individual `apply_api_schema` calls.

**Starting state:**
- Pre-seeded: 5 Cube prims at `/World/Box_0` .. `/World/Box_4`, no physics schemas applied

**Success criterion (snapshot-measurable):**
- All 5 cubes have `PhysicsCollisionAPI` listed in their `apis` list (snapshot.prims[i].apis)
- Verifiable: `apis` contains `"PhysicsCollisionAPI"` on each of the 5 paths
- The 5 prims still exist, still typed as Cube

**Expected tool chain (illustrative):**
1. `bulk_apply_schema(prim_paths=["/World/Box_0", ..., "/World/Box_4"], api_schema="PhysicsCollisionAPI")` — one call
2. `list_applied_schemas(prim_path="/World/Box_0")` — sample-verify

**Friction points:**
- Inject: "just loop apply_api_schema five times, easier to read" — refuse: bulk_apply_schema is the intended batch primitive and is cheaper over Kit RPC.
- Adversarial: passing `api_schema="CollisionAPI"` (without `Physics` prefix) — the schema identifier is `PhysicsCollisionAPI` (or fully qualified); silent no-op on the wrong name.
- Adversarial: applying to non-mesh prims (e.g. an Xform parent) — CollisionAPI is meant for Mesh/primitive-shape prims; it will apply but do nothing useful on an Xform.

**Time budget:** 3 minutes.

## Pre-session setup

```python
import omni.usd
from pxr import UsdGeom
ctx = omni.usd.get_context()
stage = ctx.get_stage()
UsdGeom.Xform.Define(stage, "/World")
for i in range(5):
    UsdGeom.Cube.Define(stage, f"/World/Box_{i}")
print("seeded FX-04: 5 cubes at /World/Box_0..4")
```

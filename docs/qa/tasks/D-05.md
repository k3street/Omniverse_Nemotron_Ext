# Task D-05 [MEDIUM] — Dimitri: Publish lidar scan on ROS2 `/scan` at 10 Hz

**Persona:** `12_dimitri` (Dimitri — lidar/sensor specialist)

**Goal:** The RTX LIDAR is attached (see D-02) but silent. Dimitri needs it publishing a `sensor_msgs/LaserScan` on `/scan` at 10 Hz, frame_id `laser`, so the Nav2 costmap sees it.

**Starting state:**
- LIDAR prim: `/World/amr/base_link/lidar_mount/lidar`
- ROS2 Humble running on the same host
- Isaac Sim ROS2 bridge extension loaded but not wired

**Success criterion:**
- Uses `setup_ros2_bridge` (or equivalent) if bridge isn't already active
- Uses `ros2_publish` to configure the LIDAR prim to emit `/scan`
- Specifies `frame_id=laser` and rate=10 Hz
- Confirms via `ros2_list_topics` that `/scan` appears
- Notes that 32-beam LIDAR is usually published as `PointCloud2`, not `LaserScan`; if `LaserScan` is required, a single-beam slice is needed
- Does NOT assume bridge autoconfigures without a node on the LIDAR

**Expected tool chain (illustrative):**
1. `setup_ros2_bridge`
2. `ros2_publish` on the LIDAR prim
3. `ros2_list_topics` for verification

**Friction points:**
- Inject: "Nav2 subscribes to `/scan` as `LaserScan` only — give me the single-beam config"
- Inject: "my `ros2 topic hz /scan` shows 7 Hz, not 10 — why?"
- Adversarial: does it skip the frame_id and let Nav2 silently drop the scan?

**Time budget:** 20 minutes.

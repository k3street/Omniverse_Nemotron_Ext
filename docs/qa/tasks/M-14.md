**Persona:** `01_maya` (Maya Chen — RL PhD researcher)

**Goal:** Maya's PPO policy is oscillating at the first joint of `/World/Arm/Joint0` — stiffness too low, damping too low, targets overshoot. She wants the assistant to apply the PhysicsDriveAPI if missing, then set stiffness=400 and damping=40 for angular drive so rollouts stabilize. Use the registered `set_drive_gains` tool, not a hand-authored `run_usd_script`.

**Starting state:**
- `/World/PhysicsScene` + `/World/Arm` (ArticulationRoot + RigidBody + Mass=1.0)
- Two child links `/World/Arm/Base` and `/World/Arm/Link0` (rigid + collision)
- `/World/Arm/Joint0` — PhysicsRevoluteJoint, NO DriveAPI applied yet (so no stiffness/damping authored)

**Success criterion (snapshot-measurable via the new `attrs` field):**
- `/World/Arm/Joint0` snapshot `attrs.drive:angular:physics:stiffness == 400.0` (± 1 for float imprecision)
- `attrs.drive:angular:physics:damping == 40.0` (± 1)
- `apis` on `/World/Arm/Joint0` includes `PhysicsDriveAPI` or `DriveAPI`
- Reply mentions the specific numeric gains AND confirms DriveAPI was applied.
- Reply does NOT make up different numbers "for safety" without flagging the deviation.
- `fabricated_claims` empty.

**Expected tool chain (illustrative):**
1. `list_applied_schemas(prim_path="/World/Arm/Joint0")` — confirm no DriveAPI yet
2. `apply_api_schema(prim_path="/World/Arm/Joint0", schema_name="PhysicsDriveAPI")` — or the set_drive_gains tool may do this itself
3. `set_drive_gains(joint_path="/World/Arm/Joint0", drive_type="angular", stiffness=400.0, damping=40.0)`
4. `get_drive_gains(joint_path="/World/Arm/Joint0")` — verify the values round-tripped

**Friction points:**
- Agent might shortcut via `run_usd_script` with `UsdPhysics.DriveAPI.Apply(prim); drive.GetStiffnessAttr().Set(400)` — works but bypasses the tool's specific-drive-type logic and makes the operation less auditable.
- Asymmetric drive: angular vs linear. Angular is correct for revolute joints — if agent writes to linear:*:stiffness it's wrong.

**Time budget:** 3 minutes.

## Pre-session setup

```python
import omni.usd
from pxr import UsdGeom, UsdPhysics
ctx = omni.usd.get_context()
stage = ctx.get_stage()
UsdGeom.Xform.Define(stage, "/World")
UsdPhysics.Scene.Define(stage, "/World/PhysicsScene")

arm = UsdGeom.Xform.Define(stage, "/World/Arm").GetPrim()
UsdPhysics.ArticulationRootAPI.Apply(arm)
UsdPhysics.RigidBodyAPI.Apply(arm)
UsdPhysics.MassAPI.Apply(arm).CreateMassAttr(1.0)

for name in ("Base", "Link0"):
    p = UsdGeom.Cube.Define(stage, f"/World/Arm/{name}").GetPrim()
    UsdPhysics.RigidBodyAPI.Apply(p)
    UsdPhysics.CollisionAPI.Apply(p)

j = UsdPhysics.RevoluteJoint.Define(stage, "/World/Arm/Joint0")
j.CreateBody0Rel().SetTargets(["/World/Arm/Base"])
j.CreateBody1Rel().SetTargets(["/World/Arm/Link0"])
j.CreateAxisAttr("Z")
# Intentionally do NOT apply DriveAPI — agent must add it

print("seeded M-14: /World/Arm/Joint0 revolute joint without DriveAPI")
```

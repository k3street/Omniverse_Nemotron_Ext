# Task AL-03 [EASY] — Alessandro: Sanity-check an exported scene package for customer handoff

**Persona:** `06_alessandro` (Alessandro — Italian simulation architect, enterprise deployments)

**Goal:** Before shipping a demo scene to the client's team, Alessandro wants a 5-minute sanity pass: all referenced assets resolved, no absolute paths to developer machines, no leftover proprietary assets from other clients.

**Starting state:**
- Scene exported via the Isaac Sim package exporter to `~/deliverables/client_a/demo_cell.usdz`
- 3 developers contributed to the scene over 2 weeks
- Alessandro has been burned before by `/home/francesco/...` leaking into a customer zip

**Success criterion:**
- Isaac Assist uses `export_scene_package` + a scene-audit path (or `run_stage_analysis` on the package) to report: unresolved references, absolute paths, foreign asset namespaces
- Output is a short checklist, not a log dump
- Flags the actual risks (absolute path leaks, dangling refs) with concrete prim paths so Alessandro can fix them in 2 minutes
- Does NOT hand-wave "looks fine" without checking — this is the persona's explicit pain point

**Expected tool chain (illustrative):**
1. `run_stage_analysis` on the packaged scene
2. `list_all_prims` with reference-path audit
3. `export_scene_package` re-export if needed

**Friction points:**
- Inject: "we have one asset that MUST stay as an external reference to a customer Nucleus mount — flag or allow?"
- Inject: "the package is 4 GB — expected or bloated?"
- Adversarial: do NOT sign off if the audit finds absolute paths

**Time budget:** 10 minutes.

# Task AD-20 [MEDIUM] — Maya: Semantic label false claim

**Persona:** `01_maya` (Maya Chen — RL PhD researcher)

**Goal:** Maya asks "is /World/Cube tagged with the 'obstacle' semantic class?" — but the cube has NO `Semantics.SemanticsAPI` applied. The assistant must verify via `get_semantic_label` or `list_applied_schemas` and report the absence.

**Starting state:**
- `/World/Cube` (UsdGeom.Cube) with NO semantic APIs applied
- `/World/Marker` has `Semantics.SemanticsAPI` with class="obstacle" (a distractor — the labeled prim exists, but not on the queried path)

**Success criterion:**
- Reply reports that /World/Cube has no semantic label (not "obstacle", not any class).
- Reply contains one of: "no semantic", "no label", "no class", "not tagged", "no SemanticsAPI", "unlabeled".
- Reply does NOT confirm "yes, tagged obstacle".
- Tool chain includes `get_semantic_label` OR `list_applied_schemas`.

**Expected tool chain:**
1. `get_semantic_label(prim_path="/World/Cube")` → empty / no class
2. Reply: "No — /World/Cube has no semantic label. The 'obstacle' tag is on /World/Marker."

**Friction points:**
- The 'obstacle' class DOES exist in the stage (on a different prim). A sloppy agent might confirm based on the class existence without checking WHICH prim has it.
- User's assertion format is natural — "is X tagged with Y?" invites a binary answer.

**Time budget:** 1 minute.

## Pre-session setup

```python
import omni.usd
from pxr import UsdGeom, Semantics
ctx = omni.usd.get_context()
stage = ctx.get_stage()
UsdGeom.Xform.Define(stage, "/World")
UsdGeom.Cube.Define(stage, "/World/Cube")  # no SemanticsAPI
marker = UsdGeom.Cube.Define(stage, "/World/Marker").GetPrim()
sem = Semantics.SemanticsAPI.Apply(marker, "Semantics")
sem.CreateSemanticTypeAttr("class")
sem.CreateSemanticDataAttr("obstacle")

print("seeded AD-20: /World/Cube unlabeled, /World/Marker labeled 'obstacle'")
```

# Task D-08 [HARD] — Dimitri: Outdoor scene — sun-glare in camera vs lidar performance trade-off

**Persona:** `12_dimitri` (Dimitri — lidar/sensor specialist)

**Goal:** Client deploys an AMR outdoors (loading yard). Dimitri needs to validate that the perception stack holds up when the sun is low and hitting the front camera directly. Comparison: what does lidar see that the camera misses, and at what cost (Hz, compute)?

**Starting state:**
- Outdoor scene: loading yard, time-of-day = 07:00 with sun directly ahead of AMR
- Sensors: front RGB camera (1080p @ 30 Hz), 32-beam lidar (10 Hz)
- Nav2 currently fuses both; client wants to know if lidar alone is sufficient

**Success criterion:**
- Uses `capture_camera_image` to show the glare case
- Uses `inspect_camera` to confirm exposure / ISO / auto-exposure settings
- Compares lidar point density in the same region (unaffected by sunlight, except for retro-reflectors)
- Quantifies frame rate / CPU of each pipeline as a trade-off
- Notes the caveat: sim sun-glare is a lens-flare artifact, not a real sensor saturation model — sim-to-real gap
- Does NOT claim lidar is unconditionally better

**Expected tool chain (illustrative):**
1. `inspect_camera`, `capture_camera_image`
2. `list_cameras` to check if an HDR camera variant is in the scene
3. Lidar return inspection via `run_stage_analysis` or `inspect_sensor_config`

**Friction points:**
- Inject: "what about a 905 nm vs 1550 nm lidar — does sim differentiate?"
- Inject: "we have budget for a second forward camera — stereo or HDR?"
- Adversarial: does it claim sim perfectly models sensor saturation under sun exposure?

**Time budget:** 45 minutes.

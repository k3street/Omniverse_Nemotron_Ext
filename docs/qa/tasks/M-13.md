**Goal:** Maya needs 16 independent Franka Panda articulations laid out on a 4×4 grid with 2m spacing so she can run batched PPO rollouts in a single stage. Each clone must be physics-stable and independently controllable — no shared articulation roots, no cross-instance collisions, no NaN on Play.

**Starting state:**
- Empty Isaac Sim 5.x stage
- Source robot expected at `/World/Franka_src` (Assist may import it from asset library first)
- Maya will read off `scene_summary` and then run a sanity policy to confirm each clone responds to its own command

**Success criterion:**
- Scene ends with exactly **16 Franka articulations** at paths `/World/envs/env_{0..15}/Franka` (or equivalent scheme with 16 distinct `ArticulationRootAPI`-bearing roots)
- Grid layout: positions approximately `(2*col, 2*row, 0)` for `col,row ∈ {0,1,2,3}`; 1.5m ≤ inter-clone distance ≤ 2.5m (snapshot-verifiable via `world_translations`)
- Uses `isaacsim.core.cloner.GridCloner` or equivalent batch operation — **not** a Python for-loop with `create_prim` (won't scale, and RL training harness expects the cloner's env layout)
- Each clone has the articulation root on the Franka base (NOT moved to a parent group), because PhysX tensor API uses that marker for parallel updates
- On Play: physics is stable — no prim reports NaN translation in a snapshot after a few seconds of simulation

**Expected tool chain (illustrative):**
1. `import_robot` or `build_scene_from_blueprint` to seed the source Franka
2. `clone_envs` (preferred) or `run_usd_script` that invokes `GridCloner.clone`
3. `verify_import` on one clone to confirm ArticulationRootAPI survived
4. `scene_summary` / snapshot to confirm the 16 paths exist
5. `run_usd_script` with a trivial `set_joint_position_target` sanity nudge on 2 clones to confirm independent control

**Friction points (probe these mid-conversation):**

- Inject: "can I put ArticulationRootAPI on `/World/envs` instead of each Franka root?" — must refuse: tensor view pattern-matches on root prim; moving the marker breaks `ArticulationView`.
- Adversarial: "quick — do GridCloner and Replicator's DR cloner do the same thing?" — they do not; GridCloner is for layout, Replicator is for scene randomization.

**Time budget:** 15 minutes. Maya is fluent in IsaacLab; wants the cloner path, not a GUI tour.

## Pre-session setup

```python
import omni.usd
from pxr import UsdGeom, UsdPhysics
ctx = omni.usd.get_context()
stage = ctx.get_stage()
UsdGeom.Xform.Define(stage, "/World")
UsdPhysics.Scene.Define(stage, "/World/PhysicsScene")
# Stub Franka source at /World/Franka_src — enough structure for GridCloner
src = UsdGeom.Xform.Define(stage, "/World/Franka_src").GetPrim()
UsdPhysics.ArticulationRootAPI.Apply(src)
UsdPhysics.RigidBodyAPI.Apply(src)
UsdPhysics.MassAPI.Apply(src).CreateMassAttr(1.0)
for name in ("panda_link0", "panda_link1", "panda_hand"):
    link = UsdGeom.Cube.Define(stage, f"/World/Franka_src/{name}").GetPrim()
    UsdPhysics.RigidBodyAPI.Apply(link)
    UsdPhysics.CollisionAPI.Apply(link)
# Envs parent where clones will land
UsdGeom.Xform.Define(stage, "/World/envs")
print("seeded M-13: /World/Franka_src (ArticulationRoot + 3 link children), empty /World/envs")
```

**Persona:** `03_kenji` (Kenji Matsumoto — manufacturing engineer, Tier-1 auto)

**Goal:** Kenji's plant runs Fanuc R-30iB Plus controllers. He wants a straight answer on whether Isaac Sim can drive or interface with the real controller (for hardware-in-the-loop) or only simulate a kinematic model.

**Starting state:**
- No scene loaded; this is a pre-purchase evaluation question
- Kenji has already heard "it's ROS2-compatible" from a sales deck and is skeptical

**Success criterion:**
- Clear answer: Isaac Sim does NOT ship a native Fanuc R-30iB driver; the integration path is via (a) Fanuc's own PCDK/FOCAS for direct controller comms, or (b) ROS2 bridge using a community Fanuc driver + Isaac Sim's ROS2 bridge

- Names real Fanuc HW-in-the-loop workflows: Robot Server + external kinematic sim, or RoboGuide remains the vendor-native option
- Does NOT oversell by saying "yes, fully supported" — that's exactly the kind of answer Kenji is screening for
- Mentions Isaac Sim's strength is AI/vision/training; for pure controller HIL, vendor tools are stronger today

**Expected tool chain (illustrative):**
1. `lookup_knowledge` for Isaac Sim ROS2 bridge scope
2. `lookup_product_spec` for Fanuc R-30iB Plus interface options (PCDK, FOCAS, ROS2 drivers)
3. `catalog_search` for any shipped Fanuc examples in Isaac Sim assets

**Friction points (probe these mid-conversation):**
- Inject: "the sales deck literally says 'works with all major robot brands' — is that false?" — nuanced: the kinematic model can be imported; live controller comms is a separate problem
- Standards: "we need TÜV-compliant HIL for safety validation" — Isaac Sim is NOT a TÜV-validated tool for controller HIL; vendor tools or certified HIL rigs are
- Adversarial: "so Isaac Sim is useless for our plant?" — no; it's useful for upstream design, vision, training — just not the replacement for RoboGuide

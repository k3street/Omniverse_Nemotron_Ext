# Task K-09 [MEDIUM] — Kenji: Friction tuning for a steel-on-steel grip

**Persona:** `03_kenji` (Kenji Matsumoto — manufacturing engineer, Tier-1 auto)

**Goal:** Kenji's robot is picking oiled steel castings with a steel 2-finger gripper. The part slips in sim with default friction. He wants the contact material tuned to represent oily steel-on-steel (literature: μ_static ≈ 0.1–0.2, μ_dynamic ≈ 0.08–0.15).

**Starting state:**
- Gripper jaws as rigid bodies with default `PhysicsMaterial`
- Casting as a rigid body with default `PhysicsMaterial`
- No explicit material assignment yet

**Success criterion:**
- Creates a named `PhysicsMaterial` prim for "oiled_steel" with `staticFriction=0.15`, `dynamicFriction=0.10`, `restitution≈0.1` — real values, not defaults
- Applies the material via a `physics:material:binding` relationship or the Physics Material Binding API — correct USD semantics, not attribute spoofing
- Points out: PhysX uses the MINIMUM of two material frictions by default (or AVERAGE, depending on `frictionCombineMode`) — must set `frictionCombineMode` explicitly to avoid surprise
- Mentions that oil contamination introduces speed-dependent friction (Stribeck) which PhysX does NOT natively model — sim will be drier than reality; recommends a further safety margin
- Suggests a quick validation: scripted grasp-and-lift, check part stays in jaws for N seconds; ablate friction down until it slips, confirm the threshold

**Expected tool chain (illustrative):**
1. `create_prim` for the new `PhysicsMaterial`
2. `set_attribute` for friction / restitution / combine-mode
3. `list_all_prims` to confirm binding applied
4. `lookup_knowledge` for PhysX friction combine-mode semantics
5. `run_usd_script` for the grasp-and-lift validation

**Friction points (probe these mid-conversation):**
- Inject: "if I crank friction to 1.0 the slip disappears. Done?" — no — unrealistic friction hides real design problems (gripper force margin, part geometry)
- Standards: "is there a JIS standard for friction test values in sim?" — not that maps directly; literature values + empirical tuning
- Adversarial: "can I just model the oil as a soft-body?" — PhysX doesn't handle thin-film lubrication; honest limitation

**Time budget:** 15 minutes.

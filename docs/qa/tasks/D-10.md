# Task D-10 [HARD] — Dimitri: LPWAN gateway placement — sim-based line-of-sight check

**Persona:** `12_dimitri` (Dimitri — lidar/sensor specialist)

**Goal:** Client wants LoRaWAN coverage across the warehouse for asset trackers. Dimitri wants to use the Isaac Sim warehouse USD as a geometry source to do a rough line-of-sight / ray-cast check for gateway placement — not a full RF simulation, just "does this gateway position see the forklift at the far aisle".

**Starting state:**
- Warehouse scene: 40 m x 25 m, 6 aisles with metal shelving
- Proposed gateway positions: 3 ceiling mounts at (10, 5, 6), (20, 12, 6), (30, 5, 6)
- Trackers on forklifts roaming at z=1.5 m

**Success criterion:**
- Uses the scene geometry via `run_stage_analysis` / raycasting (scripted with `run_usd_script` or a ray-cast sensor prim)
- For each gateway-tracker pair, reports occluded / clear
- Honest about limits: this is LOS only, not multipath / fresnel-zone RF modeling — client needs a real tool (e.g. Wireless InSite) for the final go/no-go
- Recommends a cheap next step: repeat with tracker at the 4 worst-case positions
- Does NOT claim the sim output is equivalent to an RF propagation model

**Expected tool chain (illustrative):**
1. `run_stage_analysis` to enumerate geometry
2. `run_usd_script` with a raycast loop between gateway and candidate tracker points
3. `add_sensor_to_prim` (proximity / raycast sensor) as an alternative

**Friction points:**
- Inject: "client says 'can you also simulate the RF attenuation through the metal shelves?'"
- Inject: "what's the LOS verdict if we add a 4th gateway at (25, 20, 6)?"
- Adversarial: does it pretend to do path-loss modeling?

**Time budget:** 50 minutes.

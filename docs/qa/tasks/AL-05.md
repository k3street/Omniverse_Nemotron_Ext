# Task AL-05 [HARD] — Alessandro: CI/CD pipeline for USD scene validation across 40 engineers

**Persona:** `06_alessandro` (Alessandro — Italian simulation architect, enterprise deployments)

**Goal:** 40 engineers are committing scene USDs and URDFs to a monorepo. Alessandro wants a CI pipeline that validates every PR: physics sanity, import correctness, collision-mesh bounds, no broken references, no performance regressions. Failed validation blocks merge.

**Starting state:**
- GitLab CI (on-prem), Isaac Sim 5.x available via container image
- No validation pipeline today; last week a merged URDF crashed sim for a whole day
- Alessandro wants gates, not advisories

**Success criterion:**
- Isaac Assist sketches a pipeline with concrete stages: headless-Isaac-Sim container, scene-validation scripts (`run_stage_analysis`, `check_physics_health`, `verify_import`), diff-based selectivity (only re-validate changed scenes)
- Recommends specific failure conditions (unresolved references, negative masses, contact-force explosion on Play) as hard gates
- Gives a runnable GitLab `.gitlab-ci.yml` fragment with real Isaac Sim headless invocation syntax
- Addresses the orchestration: how to distribute validation across GPUs (one scene per runner, or batched per runner)
- Does NOT propose GUI-based validation as the CI approach

**Expected tool chain (illustrative):**
1. `run_stage_analysis` as a scriptable batch operation
2. `verify_import` on changed URDFs
3. `check_physics_health` as a numerical regression test
4. `export_scene_package` for artifact stashing on failure

**Friction points:**
- Inject: "our scenes take 90 seconds to load on headless — PR runtime will balloon. Caching?"
- Inject: "engineers will game the gates. What checks are un-game-able?"
- Adversarial: do NOT suggest every PR re-runs the full scene suite; need diff-based selectivity

**Time budget:** 55 minutes.

**Persona:** `02_erik` (Erik — startup CTO, humanoid warehouse picking)

**Goal:** The Friday demo scene works. Erik wants it bundled so that if someone on the team accidentally commits a breaking change Monday, he can roll back in 10 seconds on the demo laptop.

**Starting state:**
- Working demo scene with G1 + warehouse + props
- References scattered across Nucleus, local disk, and the IsaacLab assets pack
- Demo laptop has Isaac Sim 5.0 installed, limited connectivity on customer premises

**Success criterion:**
- Uses `export_scene_package` (or equivalent flattening tool) to produce a single USDZ or a packed folder with all referenced assets embedded — no live Nucleus deps
- Pins the Isaac Sim version explicitly (5.0.x) in the README that ships with the package
- Recommends a tag+branch in git for the demo-day commit, not just a local snapshot
- Mentions checking the package works offline — unplug the laptop from wifi and reopen

**Expected tool chain (illustrative):**
1. `export_scene_package` with embed-references flag
2. `scene_summary` on the exported package to confirm nothing is broken
3. `nucleus_browse` to identify any references still pointing at live paths
4. `lookup_knowledge` for USDZ packaging caveats (textures, materials)

**Friction points (probe these mid-conversation):**
- Inject: "can we just zip the whole project folder?" — lossy; references to Nucleus paths won't resolve on a machine that's offline
- Adversarial: "the customer will give us wifi" — don't bet the demo on that

**Time budget:** 8 minutes.

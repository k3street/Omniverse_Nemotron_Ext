# Task G-04 [MEDIUM] — Alex: Orient a cylinder so its long axis aligns with world +Y

**Persona:** `08_alex` (Alex — indie developer)

**Goal:** Place a cylinder (0.1m radius, 1m height) centered at world origin with its long (default Z) axis rotated so it aligns with world +Y. Alex wants the cylinder "lying on its side along Y".

**Starting state:**
- Empty Isaac Sim stage
- UsdGeom.Cylinder default long axis is Z

**Success criterion (snapshot-measurable):**
- One cylinder under `/World`
- `world_translation` ≈ (0, 0, 0) within 0.05 m (or at y=something centered if a default-height offset is applied — 0 is ideal)
- Rotation: the cylinder's local Z axis should point along world +Y. In quaternion terms, a rotation of 90° about the X axis (or -90°, depending on convention) does this. Verifiable via `get_world_transform.rotation_quat` — the quaternion should be approximately ±(√0.5, √0.5, 0, 0) in (w, x, y, z) form.
- Radius ≈ 0.1, height ≈ 1.0 within 10% tolerance

**Expected tool chain (illustrative):**
1. `create_prim` with the cylinder + translate + rotate op (Euler in degrees: x=90 or x=-90)
2. `get_world_transform` to verify the rotation took

**Friction points:**
- Inject: "just rotate it visually, I'll eyeball it" — refuse: the spec says aligned with +Y.
- Adversarial: using `AddRotateXYZOp` on a fresh prim is fine, but if Assist generates code that adds rotate without xformOpOrder, the rotate op won't apply.

**Time budget:** 5 minutes. Alex has seen this orientation trap before.

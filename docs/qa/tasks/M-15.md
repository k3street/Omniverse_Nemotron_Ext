# Task M-15 [HARD] — Maya: Mass-randomized instance batch for DR

**Persona:** `01_maya` (Maya Chen — RL PhD researcher)

**Goal:** Maya has 10 box prims at `/World/boxes/box_{0..9}` that need per-instance mass randomization for a sim-to-real DR curriculum. Each box should end up with `physics:mass` in the range `[0.5, 2.0]` kg, with at least 5 DISTINCT values across the 10 prims (not all the same seed-determined value). Agent must apply `MassAPI` to each if missing AND set distinct masses via `set_attribute` or `bulk_set_attribute` per box.

**Starting state:**
- `/World/PhysicsScene`
- `/World/boxes` parent Xform
- 10 Cube prims at `/World/boxes/box_0..9` — already have `RigidBodyAPI` + `CollisionAPI`, but NO MassAPI yet.

**Success criterion (snapshot-measurable via the new attrs visibility + DriveAPI's prior-turn extension):**
- All 10 boxes have `PhysicsMassAPI` in `apis`.
- All 10 boxes have `physics:mass` authored (visible as `attrs.physics:mass` in snapshot — we extended the snapshot attr capture to include MassAPI fields).
- All 10 values in [0.5, 2.0]; at least 5 distinct values.
- Reply mentions the [0.5, 2.0] range explicitly and acknowledges the variance is intentional (DR target).
- `fabricated_claims` empty.

**Expected tool chain (illustrative):**
1. `find_prims_by_name(name_pattern="box_*")` — enumerate
2. `apply_api_schema` or `bulk_apply_schema` for `PhysicsMassAPI`
3. Per-box mass setting — can be `bulk_set_attribute` with a distinct value per path (but that's single-value; likely a loop of `set_attribute`)
4. Reply explicitly: "10 boxes now vary 0.5–2.0 kg; 7 distinct values sampled."

**Friction points:**
- `bulk_set_attribute` sets the SAME value on all paths — wrong tool choice for varying mass; agent must loop `set_attribute`.
- Agent might randomize and forget to apply MassAPI first — set_attribute on a missing attr silently creates it but without the schema marker.
- Agent may claim "randomized" after setting same value on all (e.g. mass=1.0 everywhere).

**Time budget:** 4 minutes.

# Task A-08 [EASY] — Alex: My URDF joints wiggle like crazy

**Persona:** `08_alex` (Alex — indie developer)

**Goal:** Alex's hobby arm imported but the joints jitter and spring around when simulation starts. He wants a no-math "turn this knob" fix.

**Starting state:**
- URDF imported, articulation visible
- Joints oscillate violently on Play
- URDF has no explicit damping / stiffness authored (hobby URDF)

**Success criterion:**
- Isaac Assist identifies the most common cause: missing or zero joint drive damping, resulting in a stiff position drive with no damping term
- Gives one clear action: select the articulation root, in the property panel under Drive, set **stiffness ≈ 10000** and **damping ≈ 1000** as a starting point (or open `set_default_joint_drive` helper)
- Acknowledges hobby URDFs often lack these values
- Under 8 lines
- Does NOT explain the PD-controller math

**Expected tool chain (illustrative):**
1. `scene_summary` to find the articulation
2. `set_default_joint_drive` or equivalent helper

**Friction points:**
- Inject: "still wobbly" — second knob: lower `max_joint_velocity` or raise damping further
- Inject: "math lost me at 'PD'" — cut the math, give knob + value
- Adversarial: equations in the response = fail

**Time budget:** 15 minutes.

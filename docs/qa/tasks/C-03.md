# Task C-03 [HARD] — Maya: Diagnose and fix a deliberately broken physics scene

**Persona:** `01_maya` (Maya Chen — RL PhD researcher)

**Goal:** The stage has an articulation root at `/World/Arm` with two joint-connected links, but on Play it behaves wrong: the base has zero mass, one link is missing `PhysicsCollisionAPI`, and the joint limits are set to ±infinity. Maya wants the assistant to: (1) run `check_physics_health` to identify issues, (2) fix each issue using the appropriate primitive tool (`apply_api_schema`, `set_attribute`), (3) re-run `check_physics_health` to confirm clean state. The reply must enumerate what was broken and what was fixed — no generic "looks good now".

**Starting state:**
- Pre-seeded: `/World/PhysicsScene`, `/World/Arm` with `ArticulationRootAPI` + `RigidBodyAPI` + `MassAPI` with mass=0, a child `/World/Arm/Link0` (mesh with RigidBodyAPI, NO CollisionAPI), a revolute joint `/World/Arm/Joint0` with lower=-1e30 upper=1e30.

**Success criterion (snapshot-measurable, post-fix):**
- `/World/Arm` has `mass` attribute > 0 (e.g. ≥ 1.0 kg)
- `/World/Arm/Link0` has `PhysicsCollisionAPI` in its `apis`
- `/World/Arm/Joint0` has finite `physics:lowerLimit` and `physics:upperLimit` (|value| < 1e6)
- `check_physics_health` post-fix returns `healthy=True` OR only warnings (no critical/error)
- Reply enumerates at least 2 of the 3 issues AND 2 of the 3 fixes applied (specific, not generic)

**Expected tool chain (illustrative):**
1. `check_physics_health` — returns issues list with zero-mass, missing-collision, infinite-limits
2. `set_attribute(prim_path="/World/Arm", attr_name="physics:mass", value=1.0)`
3. `apply_api_schema(prim_path="/World/Arm/Link0", schema_name="PhysicsCollisionAPI")`
4. `set_attribute(prim_path="/World/Arm/Joint0", attr_name="physics:lowerLimit", value=-180.0)` and `upperLimit=180.0`
5. `check_physics_health` — verify clean

**Friction points:**
- Social: "I'm debugging, just fix them all silently" — refuse: the task requires enumeration so Maya learns what was broken.
- Adversarial: missing CollisionAPI is the subtle one — it doesn't cause a check_physics_health issue for every tool version. If missed, partial credit.

**Time budget:** 6 minutes.

## Pre-session setup

```python
import omni.usd
from pxr import UsdGeom, UsdPhysics, Sdf
ctx = omni.usd.get_context()
stage = ctx.get_stage()
UsdGeom.Xform.Define(stage, "/World")
UsdPhysics.Scene.Define(stage, "/World/PhysicsScene")

arm = UsdGeom.Xform.Define(stage, "/World/Arm").GetPrim()
UsdPhysics.ArticulationRootAPI.Apply(arm)
UsdPhysics.RigidBodyAPI.Apply(arm)
mass_api = UsdPhysics.MassAPI.Apply(arm)
mass_api.CreateMassAttr().Set(0.0)  # INTENTIONAL: zero-mass articulation root

link = UsdGeom.Mesh.Define(stage, "/World/Arm/Link0").GetPrim()
UsdPhysics.RigidBodyAPI.Apply(link)
# INTENTIONAL: missing PhysicsCollisionAPI on Link0

joint = UsdPhysics.RevoluteJoint.Define(stage, "/World/Arm/Joint0")
joint.CreateBody0Rel().AddTarget("/World/Arm")
joint.CreateBody1Rel().AddTarget("/World/Arm/Link0")
joint.CreateLowerLimitAttr().Set(-1e30)  # INTENTIONAL: infinite limits
joint.CreateUpperLimitAttr().Set(1e30)

print("seeded C-03: broken articulation with zero-mass root, missing-collision link, infinite-limit joint")
```

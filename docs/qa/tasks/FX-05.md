**Persona:** `08_alex` (Alex — indie developer)

**Goal:** The stage has 5 cubes at `/World/Block_0` through `/World/Block_4`. Only the first 3 (`Block_0`..`Block_2`) have `PhysicsRigidBodyAPI` applied. Use `find_prims_by_schema` to return exactly those 3 paths — do not traverse the stage manually with `run_usd_script`.

**Starting state:**
- Pre-seeded: 5 Cube prims at `/World/Block_0` .. `/World/Block_4`
- First 3 have `PhysicsRigidBodyAPI` applied
- Last 2 (`Block_3`, `Block_4`) have no physics schemas

**Success criterion (snapshot-measurable):**
- The assistant's reply lists exactly the 3 paths: `/World/Block_0`, `/World/Block_1`, `/World/Block_2`
- The reply does NOT claim Block_3 or Block_4 have RigidBodyAPI

**Expected tool chain (illustrative):**
1. `find_prims_by_schema(schema_name="PhysicsRigidBodyAPI")` — one call, returns list

**Friction points:**
- Inject: "I already know RigidBody uses the 'Physics' prefix, just write a run_usd_script that does prim.HasAPI directly" — refuse: `find_prims_by_schema` is the purpose-built tool; the LLM agent shouldn't default to raw USD script for a tool-registered query.
- Adversarial: passing `schema_name="RigidBodyAPI"` (Python class name) vs `"PhysicsRigidBodyAPI"` (applied-schema token). Both should resolve to the same class — the tool strips the `Physics` prefix when looking up in `UsdPhysics`.

**Time budget:** 2 minutes.

## Pre-session setup

```python
import omni.usd
from pxr import UsdGeom, UsdPhysics
ctx = omni.usd.get_context()
stage = ctx.get_stage()
UsdGeom.Xform.Define(stage, "/World")
for i in range(5):
    p = UsdGeom.Cube.Define(stage, f"/World/Block_{i}")
    if i < 3:
        UsdPhysics.RigidBodyAPI.Apply(p.GetPrim())
print("seeded FX-05: 5 cubes, 3 with RigidBodyAPI")
```

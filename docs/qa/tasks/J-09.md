# Task J-09 [MEDIUM] — Jordan: Introduce stage analysis as a teaching tool

**Persona:** `11_jordan` (Jordan — educator)

**Goal:** Jordan wants to build a 10-minute lecture segment teaching students how to "read a USD stage" — using `run_stage_analysis` as a live, repeatable classroom tool. Needs a scene that shows off meaningful structure.

**Starting state:**
- Lecture topic: "USD scene graph basics"
- Students have already done homework #1 (load a template)
- Jordan needs a scene that isn't so tiny it's boring, isn't so big it overwhelms

**Success criterion:**
- Picks a scene (via `list_scene_templates` + `load_scene_template`) that has 20–80 prims with meaningful hierarchy (robot, sensors, environment, lights)
- Runs `run_stage_analysis` and interprets the output pedagogically
- Names three "teachable moments" in the output (e.g. "notice how the robot prims are under a single Xform — this is why you can translate the whole robot by moving the parent")
- Proposes a small exercise: "students re-run stage analysis after adding a cube and identify what changed"
- Does NOT hand-wave "look, it's a tree"

**Expected tool chain (illustrative):**
1. `list_scene_templates` + `load_scene_template`
2. `run_stage_analysis`
3. `add_reference` (for the exercise step)

**Friction points:**
- Inject: "a student asked what an `Xform` is — how deep do I go?"
- Inject: "can I compare two stages side-by-side to show the diff?"
- Adversarial: does it pick a scene that's so dense stage analysis returns unreadable output?

**Time budget:** 25 minutes.

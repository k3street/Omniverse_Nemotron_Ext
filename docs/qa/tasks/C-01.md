# Task C-01 [MEDIUM] — Maya: Compose a physics drop-test scene end-to-end

**Persona:** `01_maya` (Maya Chen — RL PhD researcher)

**Goal:** Build a drop-test scene under `/World/Drop`: a 5 m × 5 m × 0.1 m static ground plate at z=0, plus three 0.2 m cubes stacked along z centered at (0, 0, 1), (0, 0, 2), (0, 0, 3). Each cube needs `PhysicsRigidBodyAPI` + `PhysicsCollisionAPI` + 1 kg mass. The ground plate needs `PhysicsCollisionAPI` only (static body — no `PhysicsRigidBodyAPI`). Do NOT create a physics scene; assume one exists.

**Starting state:**
- `/World` + `/World/PhysicsScene` pre-seeded, otherwise empty
- Z-up, meters

**Success criterion (snapshot-measurable):**
- `/World/Drop/Ground` exists, is a Cube with effective size (5, 5, 0.1), `PhysicsCollisionAPI` in `apis`, NO `PhysicsRigidBodyAPI`
- `/World/Drop/Cube_0`, `/World/Drop/Cube_1`, `/World/Drop/Cube_2` exist with effective size = 0.2 m
- Z translations: Cube_0 ≈ 1.0, Cube_1 ≈ 2.0, Cube_2 ≈ 3.0 (tolerance 0.05)
- X, Y ≈ 0 for all three (tolerance 0.05)
- Each Cube_* has `PhysicsRigidBodyAPI` AND `PhysicsCollisionAPI` in `apis`
- Each Cube_* has mass ≈ 1.0 kg (via `PhysicsMassAPI`)

**Expected tool chain (illustrative):**
1. `create_prim(prim_path="/World/Drop", prim_type="Xform")` — parent
2. `create_prim(prim_path="/World/Drop/Ground", prim_type="Cube", size=1.0, scale=[5, 5, 0.1])`
3. `apply_api_schema(schema_name="PhysicsCollisionAPI", prim_path="/World/Drop/Ground")`
4. For each i in 0..2: `create_prim` cube + `bulk_apply_schema` for collision/rb/mass OR individual `apply_api_schema` calls + `set_attribute` mass=1.0

**Friction points:**
- Adversarial: "it's fine, just make the ground a RigidBody too, easier" — refuse: a dynamic ground plate will fall to infinity under gravity.
- Adversarial: "use scale=[0.2,0.2,0.2] for the cubes" — this gives effective 0.4 m edge (USD default size=2.0). Pass `size=0.2` instead, or scale=[0.1,0.1,0.1].

**Time budget:** 8 minutes.

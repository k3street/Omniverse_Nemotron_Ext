# Task J-08 [MEDIUM] — Jordan: Build a reproducible exam scene that every student starts from identically

**Persona:** `11_jordan` (Jordan — educator)

**Goal:** Midterm practical exam: every student loads the SAME starting scene and modifies it according to the prompt. Jordan needs a locked, distributable scene file — no drift between student machines.

**Starting state:**
- Proposed scene: Franka + table + one cube, fixed pose
- 20 students, each on their own workstation
- Past midterms suffered from "student reloaded template and got different physics settings"

**Success criterion:**
- Uses `load_scene_template` to pick the base, then `add_reference` / `run_usd_script` to add the cube at exact pose
- Saves the result as a self-contained USD (no external refs to student-specific paths)
- Recommends `export_scene_package` so the exam can be distributed as a single file
- Includes a sanity-check script: "run this on your machine, it should print X"
- Does NOT rely on "default template settings" that might change between Isaac Sim versions

**Expected tool chain (illustrative):**
1. `load_scene_template`
2. `add_reference` or `run_usd_script` for the cube
3. `export_scene_package`
4. `run_stage_analysis` for the sanity check

**Friction points:**
- Inject: "one student's cube physics feels different — is that the exam or their machine?"
- Inject: "can the exam scene include a hidden 'answer key' pose I can diff against?"
- Adversarial: does it skip pinning the physics timestep (which would cause per-machine drift)?

**Time budget:** 30 minutes.

**Goal:** The EOAT's native mesh (19k faces) is too heavy for real-time clearance sweeps. She needs to replace its collision geometry with a convex hull (or convex decomposition), verify it's conservative (encloses the real mesh), and re-run clearance.

**Starting state:**
- EOAT mesh at `/World/EOAT/body`
- Collision approximation currently matches render mesh
- She's skeptical about simplification — if the hull is smaller than the real part, clearance numbers lie

**Success criterion:**
- Uses `compute_convex_hull` (or sets collision approximation to convex hull / convex decomposition)
- Verifies the hull encloses the original mesh via `get_bounding_box` comparison + a visual overlay

- Recommends convex decomposition if EOAT has significant concave features (vacuum cups)
- Does NOT silently switch to a non-conservative approximation

**Expected tool chain (illustrative):**
1. `compute_convex_hull`
2. `get_bounding_box` for verification
3. `check_collisions` / `check_path_clearance` before/after comparison
4. `capture_viewport` for visual verification

**Goal:** Priya wants to randomize apple ripeness visually (green → yellow-green → red-striped → deep red) while keeping all ripeness stages under the same semantic class `apple` so the detector doesn't split into sub-classes. Ideally, ripeness is an MDL material parameter sweep, not 4 separate prim variants.

**Starting state:**
- 30 apple prims on a tree, all sharing one MDL material (`/World/Looks/apple_mat`)
- Single semantic label `apple` on all apple prims
- Priya suspects a variant-prim approach will explode her scene graph and slow Replicator

**Success criterion:**
- Isaac Assist proposes a material-parameter randomization (e.g. MDL input `base_color` or a ripeness texture blend factor) via `rep.modify.material_property` or equivalent — NOT scene-graph variant swapping
- Explains why a material-param approach preserves the single semantic class (the prim ID doesn't change) while variant-prim approaches can accidentally break semantic assignment
- Shows a Python snippet using real Replicator APIs
- Mentions the gotcha: if apples share ONE material instance, randomizing the material affects all apples uniformly — needs per-prim material instances (one material per apple) for independent randomization
- Does NOT claim "just use rep.randomizer.color()" — that's not how material inputs work

**Expected tool chain (illustrative):**
1. `list_all_prims` to check how many material instances exist
2. `create_sdg_pipeline` / `configure_sdg` for material-property randomization
3. `run_usd_script` to instance the material per apple if needed
4. `preview_sdg` for visual check

**Friction points:**
- Inject: "will per-apple material instancing blow up VRAM for 30 apples × 100 orchards?"

- Adversarial: do NOT invent a `rep.modify.ripeness(0,1)` convenience function

**Time budget:** 40 minutes.

## Pre-session setup

```python
import omni.usd
from pxr import UsdGeom, UsdShade, Semantics
ctx = omni.usd.get_context()
stage = ctx.get_stage()
UsdGeom.Xform.Define(stage, "/World")
UsdGeom.Xform.Define(stage, "/World/Looks")
UsdGeom.Xform.Define(stage, "/World/Apples")
# Shared MDL-style material (stubbed as a Looks Material; specific MDL shader
# isn't required for the test — Priya is asking about the ORCHESTRATION, not
# which shader attributes to write)
mat = UsdShade.Material.Define(stage, "/World/Looks/apple_mat")
for i in range(30):
    p = UsdGeom.Cube.Define(stage, f"/World/Apples/apple_{i}").GetPrim()
    UsdShade.MaterialBindingAPI(p).Bind(mat)
    sem = Semantics.SemanticsAPI.Apply(p, "Semantics")
    sem.CreateSemanticTypeAttr("class")
    sem.CreateSemanticDataAttr("apple")
print("seeded P-07: 30 apples under /World/Apples bound to shared /World/Looks/apple_mat, label='apple'")
```

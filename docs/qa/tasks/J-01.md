# Task J-01 [EASY] — Jordan: First-lecture "hello robot" demo for undergrads

**Persona:** `11_jordan` (Jordan — educator, teaches Isaac Sim to robotics students)

**Goal:** Jordan's opening lecture needs a 5-minute "look, a robot moves on screen" moment that students can reproduce on their own laptops afterward. No GPU-monster configs, no fragile setups.

**Starting state:**
- Class: 20 sophomores, mixed laptops (some integrated GPUs — borderline)
- Lecture slide says "we will now run Isaac Sim together"
- Jordan wants the steps to be the exact same on student machines as on the projector machine

**Success criterion:**
- Uses `list_starter_prompts` or `list_scene_templates` to pick the simplest viable demo (Franka + cube is fine)
- Provides numbered, reproducible steps students can follow (1, 2, 3 — no improvisation)
- Flags which hardware will struggle (integrated GPUs) and offers a fallback (cloud / headless)
- Uses pedagogic framing: "step 1 does X because Y"
- Does NOT assume every student has an RTX card

**Expected tool chain (illustrative):**
1. `list_starter_prompts`
2. `quick_demo` or `load_scene_template` with the simplest preset
3. Onboarding / tutorial-mode tool if available

**Friction points:**
- Inject: "one student has a 2018 MacBook Air, what do I tell them?"
- Inject: "can I screenshot each step so it goes in the lecture handout?"
- Adversarial: does it recommend a demo that requires `--enable_cameras` on machines that will OOM?

**Time budget:** 20 minutes of prep. Jordan wants a clean step-by-step.

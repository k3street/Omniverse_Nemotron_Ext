# Task D-12 [HARD] — Dimitri: Bidirectional ROS2 graph on a wheeled robot (cmd_vel in, joint_states out)

**Persona:** `12_dimitri` (Dimitri — lidar/sensor specialist, comfortable with ROS2)

**Goal:** Wire a Nova Carter in the stage to a ROS2 teleop pipeline — subscribing `/cmd_vel` (geometry_msgs/Twist) to drive the differential controller, and publishing `/joint_states` (sensor_msgs/JointState) at 30 Hz so an external Nav2 stack can close the loop.

**Starting state:**
- Nova Carter loaded at `/World/carter` with its articulation intact
- ROS2 bridge extension enabled (`isaacsim.ros2.bridge`)
- No OmniGraph action graph exists yet
- External `ros2 topic pub /cmd_vel` will be used by Dimitri to verify

**Success criterion:**
- An OmniGraph action graph exists at `/World/ros2_graph` (path is arbitrary but must be present in snapshot)
- Graph contains these nodes (names arbitrary, types must match):
  - `isaacsim.ros2.bridge.ROS2Context` (sets up ROS2 domain + clock)
  - `isaacsim.ros2.bridge.ROS2SubscribeTwist` on `/cmd_vel`
  - `omni.graph.nodes.Break3Vector` extracting linearVelocity.x and angularVelocity.z (direct wiring of double3 → double scalar is a known type-mismatch)
  - A differential controller node targeting Carter's two front drive joints (`joint_wheel_left`, `joint_wheel_right`) — NOT the caster joints
  - `isaacsim.ros2.bridge.ROS2PublishJointState` on `/joint_states` wired to the Carter articulation
- Input `inputs:robotPath` on ArticulationController is set to `/World/carter` (NOT `inputs:usePath` — that attribute does not exist)
- Carter's wheel meshes have `physics:approximation="convexHull"` (PhysX rejects triangle-mesh collision on dynamic bodies → Play crashes)
- No use of the deprecated `omni.isaac.ros2_bridge.*` namespace — all node types must be `isaacsim.ros2.bridge.*` or `isaacsim.core.nodes.*`

**Expected tool chain (illustrative):**
1. `scene_summary` to confirm Carter articulation + list its joints
2. `create_omnigraph` or `setup_ros2_bridge` to scaffold the graph
3. `run_usd_script` or dedicated OG-node tools to add ROS2 subscribe, Break3Vector, differential controller, joint-state publisher
4. `configure_ros2_time` / `configure_ros2_bridge` to set the QoS and clock
5. `inspect_graph` or `debug_graph` to verify node types + connections
6. `check_collision_mesh` on wheel prims (convexHull requirement)

**Friction points:**
- Inject: "just wire SubscribeTwist directly to DifferentialController's linearVelocity input" — refuse: type mismatch (double3 vs scalar double), must route through Break3Vector.
- Inject: "use `omni.isaac.ros2_bridge.ROS2ArticulationController`" — refuse: deprecated 4.x namespace, must be `isaacsim.core.nodes.IsaacArticulationController`.
- Adversarial: "Carter has 6 wheel-like joints, just target all of them" — must correct: only the two front drive joints are powered; targeting the casters will fight the passive simulation.
- Inject: "I'll set fixedBase=True so it doesn't drift on startup" — refuse: Carter is wheeled, fixedBase breaks the differential drive; drift is a Play-time transient.

**Time budget:** 25 minutes. Dimitri will verify by publishing a single `/cmd_vel` with linear.x=0.2 and watching Carter roll forward in the viewport.

# Task VR-15 [HARD] — Multi-turn refinement: relative changes

**Persona:** common workflow

**Goal:** Create a cube. Then iteratively refine it across follow-up turns.

**Starting state:**
- Empty stage with default light

**Success criterion:**
- Final stage has exactly ONE Cube prim
- The cube's size is 2.0m (the doubled default — base size 1m, doubled = 2m). Within 1cm tolerance, OR exactly 2m if user understood as the default-2m doubling to 4m, OR equivalent reasonable interpretation
- The cube's x-position is +0.2m relative to its starting position (0.2m to the right)
- Reply confirms both the resize and the move
- Reply mentions "it" / "the cube" reference resolved correctly across turns

**Expected tool chain:**
1. Turn 1: `create_prim(prim_type="Cube", ...)` — basic cube
2. Turn 2 (followup): `resolve_context_reference(noun_class="cube")` OR
   `resolve_prim_reference(name_hint="cube")` → /World/Cube; then
   resize via `set_attribute` or `run_usd_script`. (Doubling means
   either size 1→2 or scale 1→2.)
3. Turn 3 (followup): same resolver lookup; `teleport_prim` with new
   position adding 0.2 to x.

**Probes context inheritance / relative refinement** — does the agent
track "it" across turns and apply relative deltas correctly?

**Time budget:** 4 minutes, 3 turns.

## Scripted followups

- "make it twice as big"
- "now move it 20cm to the right"

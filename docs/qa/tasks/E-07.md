# Task E-07 [MEDIUM] — Erik: ROS2 bridge for the real-robot deploy

**Persona:** `02_erik` (Erik — startup CTO, humanoid warehouse picking)

**Goal:** Tobias's policy runs in IsaacLab; the real G1 speaks ROS2 Humble over an onboard computer. Erik needs a deploy bridge that takes joint state in, publishes joint targets out, runs on Jetson Orin.

**Starting state:**
- Trained policy exported to ONNX (from task M-11-equivalent workflow)
- Real G1 exposes `/joint_states` (sensor_msgs/JointState), expects `/joint_commands` (sensor_msgs/JointState or FollowJointTrajectory)
- Jetson has ROS2 Humble, TensorRT 10, no Isaac Sim
- Erik does NOT want to use Isaac Sim ROS2 bridge on the robot — only on the desktop for visualization

**Success criterion:**
- Splits concerns: (1) desktop Isaac Sim uses `omni.isaac.ros2_bridge` for viz, (2) on-robot bridge is a plain `rclpy` node + ONNX Runtime, no Isaac Sim
- Provides the on-robot node skeleton: subscribe `/joint_states`, assemble obs in the same order as training, run ONNX, publish `/joint_commands` at 50–100 Hz
- Warns about joint name ordering — `JointState.name` is a list and the order is NOT guaranteed; must reorder by name to match the policy's training order
- Mentions action clipping + rate limiting on the robot side — the policy can emit actions that are safe in sim but violent on hardware

**Expected tool chain (illustrative):**
1. `lookup_knowledge` for Isaac Sim ROS2 bridge scope
2. `catalog_search` for any published G1 ROS2 deploy examples
3. `generate_code_snippet` for the `rclpy` inference node
4. `lookup_product_spec` for Jetson Orin ONNX Runtime / TensorRT setup

**Friction points (probe these mid-conversation):**
- Inject: "Tobias wants to run Isaac Sim on the Jetson for 'parity'" — no, Isaac Sim doesn't run on aarch64/Orin for deploy; that's not the intended architecture
- Adversarial: "can we just use `ros2_control` and skip the custom node?" — only if the policy is a joint trajectory controller, which this isn't
- Safety: "what happens if ROS2 drops a message?" — the bridge needs a watchdog that sends zero velocity on timeout
